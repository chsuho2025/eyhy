# EY한영 AI Center 포트폴리오 시각자료 생성 프롬프트

아래 프롬프트는 게시글의 `[시각자료 01]`부터 `[시각자료 27]`까지 순서대로 대응합니다. 이미지를 생성한 뒤 파일명도 `visual-01`, `visual-02`처럼 번호를 유지하는 것을 권장합니다.

## 공통 제작 기준

- 전체 시리즈는 같은 아트디렉션을 유지합니다: 전문적인 컨설팅 리포트형 에디토리얼, 따뜻한 흰색 배경, 차콜·웜그레이 중심, 절제된 노란색 포인트, 넉넉한 여백, 정교한 그리드.
- EY의 로고·슬로건·상표·실제 사무실은 사용하지 않습니다. EY의 공식 제작물처럼 오인되는 디자인도 피합니다.
- 실제 회사 내부 화면이나 데이터를 재현하지 않습니다. 인물은 특정 실존인을 닮지 않은 일반적인 한국인 직장인으로 표현합니다.
- 이미지 안에는 긴 한국어 문장을 넣지 않습니다. 생성 모델의 문자 오류를 피하기 위해 도표는 아이콘·선·숫자 중심으로 만들고, 제목과 설명은 블로그에서 별도로 붙입니다.
- 실제 서비스 화면이 확보된 Antiframe·Chart Republic·프롬뱅크는 생성 이미지보다 실제 캡처를 우선합니다. 생성 이미지를 쓸 경우 캡션에 `개념 이미지` 또는 `공개용 재구성`이라고 표시합니다.

## 시각자료 01 — 정보 안내와 시스템 실행

**권장 비율:** 16:9 · 대표 이미지

**프롬프트:**  
전문적인 AI 컨설팅 리포트의 대표 일러스트. 중앙에 사내용 대화형 AI를 상징하는 반투명 인터페이스가 있고, 왼쪽에는 문서·규정·검색 결과를 찾아 직원에게 안내하는 흐름, 오른쪽에는 주문·휴가 신청·결제 시스템으로 이어지는 잠긴 문과 권한 열쇠 아이콘을 배치한다. ‘정보를 아는 것’과 ‘실제로 실행할 수 있는 것’의 차이가 즉시 보이게 한다. 따뜻한 흰색 배경, 차콜과 웜그레이, 작은 노란색 포인트, 정교한 아이소메트릭 에디토리얼 스타일, 실제 기업 로고와 긴 텍스트 없음, 16:9.

## 시각자료 02 — 직원 페르소나 구성도

**권장 비율:** 4:3 · 도표

**프롬프트:**  
컨설팅 보고서용 인포그래픽. 중앙의 직원 실루엣 주위에 네 개의 축이 연결된다. 직무는 노트북과 카메라 아이콘, 장소는 사무실과 외부 촬영장, 시간은 낮과 자정 시계, 질문 목적은 규정·복지·신청 아이콘으로 표현한다. 이 네 축의 조합에서 서로 다른 세 명의 직원 페르소나 카드가 만들어지는 구조. 텍스트 대신 아이콘과 숫자 1~4만 사용, 흰색 배경, 차콜·웜그레이, 노란색 포인트, 깔끔한 그리드, 4:3.

## 시각자료 03 — 정보 문의와 실행 요청 의사결정

**권장 비율:** 16:9 · 흐름도

**프롬프트:**  
기업용 AI의 의사결정 흐름도. 하나의 사용자 질문이 ‘문서 안내’ 경로와 ‘실행 요청’ 경로로 갈라진다. 실행 경로에는 시스템 연동, 사용자 인증, 권한, 필수 정보, 성공·실패 상태 확인을 뜻하는 다섯 개의 게이트가 순서대로 놓인다. 게이트를 통과하지 못하면 정직한 안내와 사람 또는 실제 시스템으로 전환되고, 모두 통과하면 제한된 실행으로 이어진다. 문자는 넣지 말고 아이콘·체크·잠금·화살표로만 표현, 전문적인 리스크 관리 도표, 흰색 배경, 노란색 포인트, 16:9.

## 시각자료 04 — 학원 질문과 업무 분기

**권장 비율:** 16:9 · 대표 이미지

**프롬프트:**  
학부모가 스마트폰 상담창에 자연어로 질문하고, 화면 중앙의 AI가 두 방향으로 결과를 나누는 전문적인 에디토리얼 이미지. 한쪽은 규정 문서와 출처를 보여주는 일반 안내, 다른 쪽은 학원 담당자에게 전달되는 검토 카드. 특정 학생 개인정보는 잠금 아이콘으로 보호된다. 친근하지만 금융·법률 서비스처럼 신뢰감 있는 분위기, 실제 학원명이나 로고 없음, 따뜻한 흰색 배경, 차콜·노란색 포인트, 16:9.

## 시각자료 05 — 규정 기반 RAG 운영 구조

**권장 비율:** 16:9 · 아키텍처

**프롬프트:**  
규정 기반 상담 AI의 전체 운영 아키텍처 인포그래픽. 왼쪽의 버전·시행일·공개 범위가 표시된 문서 더미에서 시작해, 검색과 근거 선별, 구조화된 AI 답변, 네 갈래 업무 분기, 사람 검토, 로그와 평가 대시보드로 이어진다. 각 단계가 화살표로 연결되고 최종 평가 결과가 문서와 테스트로 되돌아가는 피드백 루프를 포함한다. 긴 문구 없이 아이콘·버전 숫자·날짜 기호만 사용, 깔끔한 컨설팅 다이어그램, 흰색 배경, 16:9.

## 시각자료 06 — 5개 회귀 테스트와 4개 경로

**권장 비율:** 4:3 · 테스트 결과 도표

**프롬프트:**  
다섯 개의 테스트 질문 카드를 네 가지 결과 통로로 분류하는 회귀 테스트 인포그래픽. 결과 통로는 문서와 체크 표시, 사람 검토 아이콘, 방패와 잠금 아이콘, 빈 검색 결과 아이콘으로 구분한다. 다섯 카드가 각 기대 경로에 정확히 도착하고 모두 체크된 모습. 서비스 정확도 차트가 아니라 업무 분기 확인 도표처럼 보이게 구성, 숫자 5와 4만 명확하게, 텍스트 없음, 흰색 배경, 차콜·그레이·노란 포인트, 4:3.

## 시각자료 07 — AI가 준비하고 사람이 답하는 팬 소통
