# EY한영 AI Center 포트폴리오 도표 생성 프롬프트

## 사용 원칙

- 전체 9개 게시글에 생성 이미지는 **7개만** 사용합니다.
- 분위기를 전달하는 대표 일러스트, 인물 장면, 가상의 서비스 UI는 만들지 않습니다.
- 글만으로 관계나 순서를 이해하기 어려운 경우에만 흐름도·구성도 형태로 사용합니다.
- 하나의 도표에는 핵심 단계 3~5개만 넣습니다.
- 흰색 배경, 검은 선과 글자, 한 가지 노란색 강조만 사용합니다.
- 3D, 아이소메트릭, 사진, 캐릭터, 장식용 아이콘, 그라데이션, 그림자, 배경 패턴은 사용하지 않습니다.
- 긴 설명문은 이미지 안에 넣지 않습니다. 아래에 지정한 짧은 라벨만 정확히 사용합니다.
- 생성 뒤 글자가 틀리면 억지로 재생성하기보다 글자 없는 도표로 받고 웹에서 라벨을 추가합니다.

## 시각자료 01 — 사내용 AI의 정보 안내와 실행 요청

**사용 게시글:** 사내용 업무 AI QA
**권장 비율:** 16:9

**프롬프트:**
기업용 AI의 판단 흐름을 설명하는 매우 단순한 2개 경로 흐름도. 흰색 배경, 검은 선, 노란색 한 가지 강조색만 사용한다. 시작 노드는 ‘사용자 요청’. 여기에서 위쪽 ‘정보 문의’와 아래쪽 ‘실행 요청’으로 분기한다. 정보 문의는 ‘문서 근거 확인’ 다음 ‘안내’로 끝난다. 실행 요청은 ‘연동·인증·권한·필수 정보·상태 확인’이라는 하나의 검증 박스를 거친다. 조건을 충족하면 ‘제한적 실행’, 충족하지 못하면 ‘한계 안내와 실제 처리 경로’로 끝난다. 총 노드 수를 최소화하고 화살표 방향을 명확하게 한다. 인물, 로봇, 사무실, 가상 UI, 장식 아이콘, 3D 효과, 그라데이션, 그림자 없음. 한국어 라벨은 지정한 짧은 문구만 정확히 사용. 전문 컨설팅 보고서용 벡터 다이어그램, 16:9.

## 시각자료 03 — 팬 활동 데이터 최소화

**사용 게시글:** 베리즈 Gemini API 팬 소통 AI
**권장 비율:** 16:9

**프롬프트:**
입력 데이터가 단계적으로 줄어드는 단순 수평 파이프라인 도표. 흰색 배경, 검은 선, 노란색 강조만 사용한다. 왼쪽부터 ‘전체 공개 활동’, ‘중복·저정보 제거’, ‘위험 활동 분리’, ‘현재 대화의 최소 근거’, ‘아티스트 최종 확인’의 다섯 박스를 화살표로 연결한다. 첫 박스는 넓고 데이터 조각이 많으며 네 번째 박스로 갈수록 박스 안의 데이터 조각이 줄어드는 방식으로 최소화를 표현한다. 위험 활동은 본 흐름 아래의 별도 작은 박스로 빠지게 한다. 팬이나 아티스트 인물, 플랫폼 화면, 말풍선 장면, 로봇, 3D, 그라데이션, 그림자 없음. 한국어 라벨은 지정 문구만 정확히 사용. 데이터 거버넌스 보고서용 벡터 다이어그램, 16:9.

## 시각자료 04 — TTS 발음 데이터 처리와 사람 검토

**사용 게시글:** 음악 TTS 발음사전 데이터
**권장 비율:** 16:9

**프롬프트:**
숫자와 분기 관계가 한눈에 보이는 단순 데이터 처리 흐름도. 흰색 배경, 검은 선, 노란색 강조만 사용한다. 왼쪽부터 큰 숫자 ‘35,000 후보’, 다음 ‘3,500건 수동 조사’, 다음 ‘패턴·예외 분류’를 배치한다. 마지막은 위쪽 ‘반복 패턴 → 자동화 후보’와 아래쪽 ‘조어·출처 충돌 → 사람 검토’로 두 갈래 분기한다. 가장 중요한 숫자 35,000과 3,500을 크게 표시하고 나머지는 짧은 라벨만 쓴다. 헤드폰, 음표, 인물, 아티스트 이미지, 장식 아이콘, 3D, 그라데이션, 그림자 없음. 숫자와 한국어 라벨을 정확히 표기. 전문 데이터 분석 보고서용 벡터 다이어그램, 16:9.

## 시각자료 07 — Antiframe 생성·검토·렌더 흐름

**권장 비율:** 16:9

**프롬프트:**
AI 영상 제작 시스템을 다섯 단계로 축약한 단순 시스템 흐름도. 흰색 배경, 검은 선, 노란색 강조만 사용한다. 왼쪽부터 ‘대본 구조화’, ‘음성·이미지·영상 생성’, ‘사용자 검토’, ‘렌더 큐·외부 워커’, ‘최종 영상’의 다섯 박스를 화살표로 연결한다. ‘사용자 검토’에서 두 번째 단계로 돌아가는 작은 재생성 화살표를 하나 넣고, ‘렌더 큐·외부 워커’ 아래에 작은 상태 라벨 ‘대기·진행·실패·완료’를 넣는다. ## 생성 이미지를 사용하지 않는 게시글

### 프롬뱅크

생성 이미지를 사용하지 않습니다. 공개 가능한 실제 서비스 화면을 확보하면 카드 썸네일 또는 본문 증빙으로 한 장만 사용합니다. 8개월간 422개의 월별 누적 데이터가 확인되지 않았으므로 임의의 성장 그래프를 만들지 않습니다.

### 온보딩 개선

생성 이미지를 사용하지 않습니다. 의도·실제 경험·차이·제안은 본문의 HTML 표와 단계 목록만으로 설명합니다. 회사 내부 정보를 공개하지 않으면서 사용할 실제 자료가 확보될 때만 익명화한 증빙을 추가합니다.
