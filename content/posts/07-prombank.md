# 422개의 프롬프트 템플릿을 운영한 공유 플랫폼 프롬뱅크

> **개인 프로젝트 · 생성형 AI 활용 · 서비스 제작·8개월 운영**

생성형 AI에 대한 관심을 개인적인 사용에서 멈추지 않고 프롬프트 공유 플랫폼 `프롬뱅크`를 제작해 8개월간 운영했습니다. 422개의 템플릿을 축적하며 한 번 잘 작동한 지시를 다른 사람도 다시 사용할 수 있는 자료로 바꾸는 문제를 다뤘습니다.

> **발행 전 확인**
> 이 글은 서비스 URL·화면·저장소와 본인의 기획·개발·운영 범위를 확인한 뒤 완성해야 하는 조건부 초안입니다. 현재 확인된 성과는 `8개월 운영`과 `422개 템플릿`이며 사용자 수나 활용 효과는 아직 사용하지 않습니다.

## 핵심 요약

- 흩어진 프롬프트를 목적에 따라 찾고 다시 사용할 수 있는 공유 공간을 직접 만들었습니다.
- 한 문장을 복사하는 자료보다 사용 목적, 바꿔 넣을 입력과 기대 결과가 드러나는 템플릿이 필요하다는 문제를 다뤘습니다.
- 장기 운영 경험은 분명하지만 기능·기여 범위와 사용자 효과는 증빙 확인 후 보완해야 합니다.

## 프로젝트 소개: 잘 작동한 프롬프트는 왜 다시 쓰기 어려울까요?

생성형 AI를 사용하면서 같은 목적의 작업도 입력 맥락과 출력 조건에 따라 결과가 크게 달라지는 점에 관심을 가졌습니다. 한 번 잘 작동한 프롬프트를 개인 메모에 저장해도 시간이 지나면 어떤 상황에서 사용했는지 잊거나, 다른 사람이 그대로 복사했을 때 필요한 입력을 알기 어려웠습니다.

저는 이 문제를 다음과 같이 정의했습니다.

> 프롬프트를 문장 모음이 아니라 목적에 맞게 찾고 필요한 부분을 바꿔 다시 실행할 수 있는 템플릿으로 관리할 수 있을까?

개인적인 관심을 생각으로만 남기지 않고 프롬프트 공유 플랫폼 `프롬뱅크`를 제작했습니다. 8개월 동안 422개의 프롬프트 템플릿을 축적·공유했습니다.

## 제가 직접 맡은 범위를 공개 전 확정하겠습니다

`서비스를 제작했다`는 문장만으로는 기획, 개발, 콘텐츠 작성과 운영 가운데 무엇을 직접 했는지 알 수 없습니다. 아래 표는 제출 전에 실제 저장소와 운영 자료를 기준으로 채워야 합니다.

| 영역 | 직접 수행한 내용 | 필요한 증빙 |
|---|---|---|
| 문제 정의·기획 | **[사실 확인 후 보완]** | 초기 기획 문서·메모 |
| 화면·기능 개발 | **[사실 확인 후 보완]** | 저장소·배포 화면 |
| 템플릿 작성·분류 | **[사실 확인 후 보완]** | 템플릿 목록·대표 사례 |
| 콘텐츠 운영 | 8개월간 422개 템플릿 축적·공유 | 등록 날짜·운영 기록 |

이 확인이 끝나기 전에는 모든 기능과 422개 템플릿을 혼자 개발·작성했다고 단정하지 않습니다.

## 프롬프트를 재사용 가능한 템플릿으로 바꾸는 구조

이 섹션은 실제 프롬뱅크 템플릿을 확인한 뒤 대표 사례로 교체합니다. 공개할 사례에서는 적어도 다음 요소가 보여야 합니다.

```text
사용 목적: 이 프롬프트로 해결하려는 일
입력 변수: 사용자가 상황에 맞게 바꿔 넣을 정보
고정 조건: 역할·제약·금지 사항
출력 형식: 표·목록·문장 등 기대하는 결과 구조
사용 예시: 어떤 입력과 결과가 나오는지
검토 사항: 사람이 사실과 적절성을 확인할 부분
```

단순히 긴 프롬프트를 제공하는 것보다 사용자가 바꿔야 하는 정보와 고정해야 할 조건이 구분돼야 다시 사용할 수 있습니다. 특히 결과가 사실을 포함한다면 출처 확인이나 사람의 검토가 필요하다는 안내도 함께 있어야 합니다.

위 문단은 프롬뱅크의 실제 템플릿 구조를 확인해 일치할 때만 수행 사실로 전환합니다. 일치하지 않으면 `운영 경험에서 현재 다시 설계한 개선안`으로 라벨을 바꿉니다.

## 한 번 만든 뒤 8개월 동안 422개를 쌓았습니다

짧은 데모와 운영되는 서비스의 차이는 새 콘텐츠를 계속 추가하고 기존 자료를 다시 정리하는 과정에 있습니다.

이 글에서는 템플릿 수만 성과로 제시하지 않고 실제 자료를 확인해 다음 변화를 추가할 예정입니다.

- 초기 분류와 운영 후 분류가 어떻게 달라졌는가
- 중복되거나 다시 사용하기 어려운 템플릿을 어떻게 다뤘는가
- 설명이나 입력 예시를 수정한 대표 사례가 있는가
- 꾸준히 자료를 등록하고 검수한 운영 방식은 무엇이었는가

기록으로 확인되지 않는 개선 효과는 추가하지 않습니다. 422개라는 수치는 사용자가 실제로 업무 시간을 줄였다는 증거가 아니라, 아이디어를 서비스와 지속적인 콘텐츠 운영으로 옮긴 실행의 규모를 보여주는 자료입니다.

## 결과와 한계

생성형 AI 프롬프트 공유 플랫폼을 제작해 8개월 동안 운영했고, 422개의 템플릿을 축적·공유했습니다. 생성형 AI에 대한 관심을 서비스 제작과 지속적인 운영으로 연결한 경험입니다.

현재 사용자 수, 재방문율, 템플릿 실행 횟수와 업무 효과는 확인되지 않았습니다. 서비스 URL, 실제 운영 기간, 템플릿 목록과 제 기여 범위도 제출 전 원본 자료로 다시 확인해야 합니다.
