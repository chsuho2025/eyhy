# 공개 차트 여섯 개로 K-pop 트렌드 차트를 만든 과정

> **개인 프로젝트 · 공개 데이터 수집·가공 · 분석 서비스 운영**  
> **활동기간: 2026년 3월–현재**

카카오엔터테인먼트에서 접한 10·20대 이용자와 음악 관심 확대라는 고민을 개인적인 데이터 가설로 바꿨습니다. Apple Music·Spotify·YouTube의 공개 순위를 조합하고 가중치, 날짜별 Snapshot과 관리자 검토를 포함한 `Chart Republic`을 만들었습니다.

## 핵심 요약

- 실제 연령별 청취 데이터가 없는 한계를 밝히고, 바이럴·지역·숏폼 공개 순위를 대체 신호로 사용했습니다.
- 출처별 점수를 조합해 후보 50곡과 TOP 25를 만들고, 날짜별 변화를 다시 계산할 수 있게 저장했습니다.
- AI 모델 프로젝트는 아니며, 불완전한 외부 데이터를 검증 가능한 데이터 제품으로 운영한 경험입니다.

> **[시각자료 13 삽입 위치]**  
> 여러 공개 음악 차트가 하나의 Chart Republic 트렌드 화면으로 모이는 대표 이미지

---

## 비즈니스 고민을 검증 가능한 가설로 바꿨습니다

근무 중 10·20대 이용자를 늘리고 멜론에 대한 관심을 높이는 방법을 고민한다는 이야기를 접했습니다. 공식 과제로 맡은 일은 아니었지만, 전체 이용량 중심의 차트와 별도로 바이럴과 숏폼에서 빠르게 확산되는 곡을 보여주면 젊은 이용자가 새로운 음악을 발견하는 데 도움이 될 수 있다고 생각했습니다.

초기 가설은 다음과 같았습니다.

> 국내 스트리밍 순위와 지역·바이럴·숏폼 반응을 함께 보면 빠르게 떠오르는 K-pop 흐름을 더 일찍 보여줄 수 있다.

그러나 제게는 실제 연령별 청취 데이터가 없었습니다. 공개 차트만으로 `10·20대가 가장 좋아하는 음악`을 증명할 수도 없었습니다. 그래서 대상을 단정하는 표현 대신 공개 데이터를 대체 신호로 사용하는 개인 실험으로 범위를 좁혔습니다.

## 여섯 개 공개 차트의 역할을 구분했습니다

같은 순위라도 각 출처가 보여주는 행동은 다릅니다. 모든 차트를 단순 합산하지 않고 어떤 신호를 반영하려는지 먼저 정했습니다.

| 공개 데이터 | 반영하려는 신호 |
|---|---|
| Apple Music Top 100 Korea | 국내 전체 음악 소비 흐름 |
| Apple Music Top 25 Seoul | 도시 단위의 빠른 반응 |
| Spotify Top 50 Korea | 국내 Spotify 청취 흐름 |
| Spotify Viral Hits Korea | 공유와 바이럴 확산 신호 |
| YouTube Music Weekly | 영상·음악 소비의 주간 흐름 |
| YouTube Shorts Daily | 숏폼에서 사용되는 음악 반응 |

순위가 높을수록 큰 점수로 변환하고 출처별 가중치를 적용했습니다. 새로운 반응을 빨리 포착하려는 목적에 맞춰 바이럴·지역·숏폼 신호에 상대적으로 높은 비중을 두었습니다.

> **[시각자료 14 삽입 위치]**  
> 스트리밍·도시·바이럴·주간 영상·숏폼의 여섯 신호가 하나의 종합 점수로 모이는 도표

이 가중치는 실제 10·20대 행동으로 검증한 모델이 아닙니다. 서비스가 무엇을 더 중요하게 볼 것인지 제가 설정한 초기 가설이며, 추후 사용자 데이터로 수정해야 할 값입니다.

## 수집에서 발행까지 반복 가능한 흐름을 만들었습니다

```text
6개 공개 차트 수집
→ 곡 ID·제목·아티스트 정규화
→ 같은 곡의 출처별 순위 결합
→ 후보 50곡 구성
→ 편집 검토 점수 반영
→ 가중치 점수와 순위 계산
→ 이전 날짜 Snapshot과 비교
→ TOP 25 공개
```

Node.js 스크립트로 동일 곡을 묶고 종합 점수를 계산했습니다. 날짜별 JSON Snapshot을 보존해 전일 순위, 최고 순위, 차트 진입 일수와 최근 7일 기록을 만들었습니다.

검증 스크립트에서는 다음 항목을 확인했습니다.

- 기준 날짜와 필수 필드 누락
- 중복 곡과 Track ID 일관성
- 이전 순위·최고 순위·진입 일수의 관계
- 음악과 앨범 이미지 링크 형식

> **[시각자료 15 삽입 위치]**  
> 공개 데이터 수집부터 정규화·점수화·Snapshot·관리자 검토·TOP 25 발행까지의 운영 흐름도

## 자동 수집 뒤에 관리자 검토를 남겼습니다

공개 데이터만으로 동명이곡, 잘못 연결된 앨범 이미지와 비정상적인 후보를 모두 해결할 수 없었습니다. 자동 점수만 믿으면 데이터 연결 오류가 높은 순위로 노출될 수 있었습니다.

후보 50곡의 편집 검토 점수를 수정하고 예상 순위를 확인한 뒤 최종 JSON을 내보내는 관리자 화면을 만들었습니다. 앨범 이미지 후보는 제목과 아티스트 일치도를 점수화해 연결하고, 기준보다 낮으면 잘못된 이미지를 억지로 붙이지 않게 했습니다.

사람의 검토가 계산 결과를 임의로 바꾸는 단계가 되지 않도록 원본 순위, 자동 계산 값과 편집 값을 함께 확인할 수 있는 구조를 지향했습니다.

## 보여주고 싶은 주장보다 데이터가 말할 수 있는 범위를 선택했습니다

서비스는 한국어와 영어 UI로 현재 순위, 전일 대비 변화, 최고 순위, 진입 일수와 최근 7일 추이를 보여줍니다.

처음의 관심은 10·20대였지만 실제 연령별 데이터가 없기 때문에 최종 공개 표현은 다음처럼 수정했습니다.

> 국내 스트리밍·바이럴·숏폼 신호를 조합한 K-pop 트렌드 차트

이는 작은 문구 수정이 아니라 분석의 증거 수준과 결론의 범위를 맞추는 과정이었습니다.

## 결과와 한계

공개 차트 여섯 개, 후보 50곡과 TOP 25, 날짜별 Snapshot, 관리자 검토 화면과 검증 스크립트를 포함한 데이터 운영 MVP를 구현했습니다.

이 프로젝트는 AI 모델을 사용한 Case가 아닙니다. 실제 연령별 청취 데이터, 사용자 반응과 멜론 관심도 변화도 확인하지 못했습니다. 외부 차트의 정책과 제공 방식이 바뀌면 수집 구조도 함께 점검해야 합니다.

다음 단계에서는 트렌드 차트에 대한 사용자 인터뷰와 클릭·저장·재생 같은 행동 데이터를 확보해 대체 신호와 가중치가 실제 발견 경험에 도움이 되는지 검증해야 합니다.

## 이 경험이 남긴 기준

분석 결과를 신뢰하려면 출처가 다른 데이터를 같은 의미로 취급하지 않고, 수집 오류와 사람의 판단을 다시 확인할 수 있어야 합니다. 원하는 결론보다 현재 데이터가 말할 수 있는 범위를 먼저 밝혀야 합니다.

Chart Republic은 현업에서 들은 고민을 개인적인 가설로 바꾸고, 공개 데이터 수집·가공·검증과 운영 화면까지 직접 만든 실행 경험입니다.
